# V5 開發紀錄 (Development Record)

## 專案標題：採購智能分析平台 V5.0

### 開發目標
在 V4 的 Excel 互動基礎上，實作「高品質 PDF 報告匯出」與「全方位行動裝置 (RWD) 優化」，確保決策者能隨時隨地從各種終端獲取清晰的分析報告。

### 版本演進

| 版本 | 發布時間 | 核心特色 |
|------|----------|----------|
| V1 | 2024-Q3 | 基礎 8 頁面，深色玻璃擬態 |
| V2 | 2024-Q4 | 新導航架構，年度/品類篩選器，Plotly 圖表 |
| V3 | 2025-Q2 | Vizro 現代化框架，Plotly 圖表 |
| V4 | 2025-Q4 | Chart.js + SheetJS，Excel 上下傳，12 頁整合 |
| **V5** | **2026-Q1** | **PDF 報告匯出，行動裝置優化 (RWD)，主題相容性修正** |

### 採用技術棧 (新增)
- **PDF 匯出**：html2pdf.js 0.10.1 (封裝 html2canvas + jsPDF)
- **RWD 架構**：原生 CSS Grid + Flexbox + Media Queries
- **主題控制**：`data-theme` 屬性驅動的 CSS 變數系統

### V5 開發歷程

#### Phase 1：行動裝置優化 (RWD)
- **導航列適應**：在寬度小於 768px 時，側邊欄自動縮減為圖示模式，或移動至頂部。
- **頂部工具列優化**：手機版隱藏按鈕文字，僅保留圖示以節省空間；縮減 Padding 與字級。
- **佈局調整**：將原本的三欄/兩欄 Grid 在移動端強制轉為單欄垂直排列，確保圖表可讀性。
- **觸控優化**：拉大按鈕點擊區域，防止誤觸。

#### Phase 2：PDF 匯出功能與透明度問題修復 (核心挑戰)
- **初步實作**：引入 `html2pdf.js`，實作單一頁面匯出為 A4 橫式 PDF。
- **透明度問題分析**：發現 html2canvas 在克隆 DOM 渲染時，無法處理 CSS 變數中的 `rgba()` 半透明色與 `backdrop-filter`，導致報告整體灰暗透明。
- **終極修復策略**：
    1. 在 `onclone` 回調中注入「PDF 專用覆寫 CSS」。
    2. 強制將所有半透明色（如 `--glass`, `--border`）替換為針對主題設計的**不透明實色**。
    3. 全域停用 `animation`、`transition` 與 `backdrop-filter`，避免擷取到動畫起始的 0 透明度瞬間。
    4. 修正 `onclone` 執行時 CSS 變數解析失敗的問題。

#### Phase 3：主題相容性修正
- 修改 `toggleTheme` 邏輯，確保主題切換與圖表顏色（Chart.js）同步更新。
- 確保 PDF 匯出時能精確辨識當前屬性（Light/Dark），並正確套用對應的實色覆寫表。

### 重大技術決策

1. **PDF 實色覆寫 (Solid Color Override)**：放棄在 PDF 中保留玻璃擬態透明效果，改用高品質的實色渲染，換取最高的列印可讀性。
2. **免 Server RWD**：維持單一 HTML 檔案結構，所有響應式邏輯純靠 CSS 定義。
3. **html2canvas 效能取捨**：將 `scale` 設定為 2 以確保圖表清晰度，並使用 `jpeg` 壓縮優化檔案大小。

### 已知限制
- PDF 匯出目前採雙倍解析度，若單頁圖表過多，生成時間約需 3-5 秒。
- iPhone 部分瀏覽器可能因記憶體限制無法產出超大 PDF，建議使用桌面版 Chrome。
